3. 任务分解规则

系统会将PRD解析为以下任务结构：

第一阶段：设定完善

1. 世界观细化（2-3天）
   - 地理设定
   - 力量体系规则
   - 社会结构文档
2. 人物档案建立（3-4天）
   - 主角详细设定
   - 重要配角设定
   - 关系图谱绘制

第二阶段：大纲构建

3. 整体故事线梳理（2天）
   - 三幕结构规划
   - 关键转折点设计
   - 悬念布置方案
4. 分卷大纲（每卷1天）
   - 卷名与核心冲突
   - 章节列表（含标题和概要）
   - 情绪曲线设计

第三阶段：章节写作

5. 第1-10章写作（每天2章）
   - 开篇吸引力检查
   - 金手指展示
   - 第一个小高潮
6. 第11-30章写作（每天3章）[...]

第四阶段：质量管控

7. 每卷完成后的自查
   - 情节逻辑检查
   - 人物一致性检查
   - 节奏把控
8. 读者反馈分析
   - 评论关键词提取
   - 读者期待分析
   - 后续章节调整

5. 模型分工策略

主模型 (Claude Sonnet)：
- 核心情节设计
- 人物深度塑造
- 复杂世界观构建
- 关键章节写作

研究模型 (Perplexity)：
- 历史/文化背景研究
- 专业知识核实
- 流行趋势分析
- 读者偏好调查

兜底模型 (Gemini Flash)：
- 日常情节填充
- 对话润色
- 场景描写补充
- 过渡段写作

6. 复杂度评估指标

评估维度：
1. 世界观复杂度 (1-5分)
2. 人物数量与深度 (1-5分)
3. 情节线数量 (1-5分)
4. 专业知识需求 (1-5分)
5. 更新强度 (1-5分)

关键路径识别：
- 世界观设定 → 人物设定 → 主线设计 → 分卷规划
- 黄金三章 → 第一个高潮 → 中期转折 → 结局

8. 异常处理机制

常见问题与解决方案：
1. 卡文状态：
   - 触发条件：同一任务停滞超过2天
   - 处理：启用"情节救援"工具，提供3个发展方向

2. 设定矛盾：
   - 触发：系统检测到世界观不一致
   - 处理：启动"设定协调"会话，自动修正冲突

3. 节奏失衡：
   - 触发：连续3章爽点密度<0.5
   - 处理：插入"小高潮生成"任务

4. 读者反馈负面：
   - 触发：负面评论比例>30%
   - 处理：启动"读者预期调整"分析

使用流程示例

1. 初始化：用户按照PRD模板填写小说构想
2. 任务生成：AI解析生成详细任务列表，评估复杂度
3. 智能调度：每天询问"今天写什么？"，获得最优任务
4. 协作写作：
   - 主模型负责关键情节
   - 研究模型补充专业知识
   - 兜底模型处理过渡内容
5. 质量控制：每完成5章自动进行一致性检查
6. 灵活调整：根据读者反馈动态调整后续大纲

优势特点

1. 结构化管理：将创作过程模块化，降低心理负担
2. 节奏控制：确保爽点分布均匀，避免节奏失衡
3. 一致性保障：自动检查设定矛盾，维护作品统一性
4. 数据驱动：基于读者反馈优化后续内容
5. 灵活应变：支持中途修改主线，自动调整后续任务

这个skill特别适合：

- 长篇连载作者，需要保持稳定更新
- 多线叙事作品，需要管理复杂情节
- 设定严谨的硬核小说，需要维护世界观一致性
- 尝试新题材，需要大量前期研究的作者

用户只需专注在PRD中描述"我想写一个什么样的故事"，系统会自动处理"如何分解任务、按什么顺序写、如何保持质量"等项目管理问题。
