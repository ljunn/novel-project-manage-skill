# 大纲修整与二次校验

用于主线已经有了，但总觉得大纲发虚、像流水账、扩写仍会卡文，或人设和故事没有真正咬住的时候。

默认定位：

- 它是二次校验层，不替代 `references/planning/main-plot-construction.md`
- 顺序仍然是：主角目标 -> 起爆事件 -> 连锁反应 -> 逆推问题链 -> 大纲修整
- 只有主线发动机先成立，这份文档才有意义

## 1. 这份文档解决什么问题

常见信号：

- 事件不少，但彼此只是排队出现，没有明显因果
- 人设标签很多，但换个人来演也差不多
- 立意写得很好听，但正文里落不到选择、代价和后果上
- 按大纲扩写还是卡住，因为只有“接下来发生什么”，没有“这一场怎么打起来”
- 章节能概括成一句话，但拆不到具体场景和关系变化

## 2. 大纲不是事件流水账

一个能用来扩写的大纲，至少要回答：

- 关键节点是什么
- 谁想做什么，为什么是现在
- 谁或什么在阻止
- 主角因此付出什么代价，或暴露什么缺口
- 关系、立场、认知或局势因此改变了什么
- 这一轮结束后，前台还剩什么问题

如果大纲只能写出“先发生 A，再发生 B，最后发生 C”，那还只是事件提要，不是可执行大纲。

## 3. 立意、人设、故事要咬成一条链

更稳的检查顺序：

兴趣或情绪来源 -> 核心命题 -> 主角缺口 -> 人设行动逻辑 -> 核心冲突 -> 关键关系 -> 故事推进 -> 平台看点

逐项追问：

- 你真正想写的，是哪种情绪、刺痛、欲望或反差
- 这个兴趣最后沉淀成什么命题
- 什么样的人最适合承受这个命题，而不是只负责听作者发言
- 这个人的缺口、执念或不完美，会怎么把故事越推越偏
- 他会和谁形成镜像、互补、对照、误解或互相拯救
- 这种人物配置，具体会逼出什么故事，而不是停在设定描述
- 这个故事对平台读者来说，最稳定的期待点是什么

如果“命题”和“人设”之间没有摩擦，大纲通常会空心。

## 4. 最常见的四种失手

### 4.1 立意只存在于说明文字里

错误征兆：

- 文档里写了“这本书要表达什么”，正文却主要靠角色讲道理
- 关键情节并没有逼角色在价值、关系、利益之间做选择

修法：

- 把命题改写成选择
- 把选择改写成代价
- 把代价改写成后果

### 4.2 人设是孤立标签

错误征兆：

- 角色被写成“高冷、疯批、善良、虚荣”，但剧情不依赖这些特质
- 换个性格，情节几乎不需要改

修法：

- 追问这个性格会让他更容易做错什么
- 追问这个缺口会在哪个冲突里被放大
- 追问什么关系最能照出这个人的盲区

### 4.3 简纲只有事件，没有对抗

错误征兆：

- 能说清发生了什么，说不清谁在争什么
- 每个节点都像“播报剧情”，不是“逼出行动”

修法：

- 每个节点至少补上行动目的、冲突点、关系变化
- 补不出来的节点，优先合并、删减或重做

### 4.4 按大纲扩写仍卡文

错误征兆：

- 知道下一章写什么事，但不知道场景怎么展开
- 总想靠解释、回忆或旁白把剧情糊过去

修法：

- 不再只写“事件”，改写成“关键场景”
- 每场先定谁要什么、谁来拦、拦住后会失去什么
- 先把最硬的一场定出来，再倒推铺路场景

## 5. 线性大纲的实用写法

线性大纲仍然是默认选项，因为最利于连载推进、复盘和同步项目记忆。

但线性不等于流水账。每个关键节点建议至少写出：

- 事件：发生了什么
- 目的：谁想要什么
- 阻碍：谁或什么不让他拿到
- 关键场景：这一段最该写成什么场面
- 变化：关系、认知、身份、局势变了什么
- 钩子：这一轮之后，新的前台问题是什么

## 6. 思维导图和视觉整理怎么用

可用，但要降级成辅助工具：

- 思维导图：适合整理人物关系、势力站位、伏笔分叉和规则关联
- 图像整理：适合作为角色气质、场景氛围和时代质感参考

不要让它们替代文本大纲。真正要同步进项目记忆、章节计划和后续续写的，仍然应落回线性文本。

## 7. 张纲和场景纲的最低颗粒度

单章扩写前，至少把每个关键帧写到下面这个程度：

- 关键事件
- 行动目的
- 冲突点
- 关键场景
- 关系或认知变化
- 回报或钩子

如果某一章只能写出“发生了一件事”，却写不出“为什么会打起来、为什么值得写、谁因此改变了什么”，这一章通常还不能直接扩正文。

## 8. 与本仓库工作流的衔接方式

默认做法：

1. 先用 `references/planning/main-plot-construction.md` 拎直主线
2. 再用 `references/planning/outline-template.md` 建立项目总纲，并用 `references/planning/chapter-plan-template.md` 建立章节规划
3. 发现大纲空心、发散或扩写卡住时，再用这份文档回查
4. 真正进入单章执行时，接 `references/chapter/chapter-workflow.md`

不要反过来把这份文档抬到最前面。先有发动机，再谈修整手感。
