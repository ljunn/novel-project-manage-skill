# 章节规划

用于承载待办事项、章节规划表、章节摘要和进度统计；全书主线、终局锚点和长期结构约束统一写入 `docs/项目总纲.md`。

填写原则：

- 立项初期只需先搭 `待办清单` 和 `章节规划` 表的前几章
- 开始写某章前，再补这章的关键事件、钩子和必要摘要
- 不要求开局一次铺满几百章，只要求当前 3-10 章可执行

## 张纲关键帧检查

`章节规划` 表格保持简版字段，用于项目总览。真正扩写前，每一章或每个关键节点至少额外补齐下面 6 项：

- **关键事件**：[这一章真正发生了什么]
- **行动目的**：[谁想拿到、守住、证明、隐瞒或逃离什么]
- **冲突点**：[谁或什么在直接阻止]
- **关键场景**：[最能兑现本章卖点、情绪变化或局势变化的场面]
- **关系/认知变化**：[人物关系、立场、信任或判断发生了什么变化]
- **回报/钩子**：[本章给读者什么回报，并把什么问题送进下一章]

扩写前自查：

- [ ] 删掉这一章后，主线、人物或关系会明显受损，而不是几乎没差
- [ ] 这一章不只是“发生了一件事”，而是逼出了选择、代价或误判
- [ ] 这一章至少有一个场景同时服务情节推进和人物/关系塑造
