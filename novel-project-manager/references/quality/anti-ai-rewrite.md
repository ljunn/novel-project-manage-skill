# 去 AI 味返修卡

用于把 `review / lint / dialogue-pass` 命中的语言问题，转成可执行的返修动作，而不是只知道“有 AI 味”。

默认目标：

- 保信息
- 保因果
- 保章节钩子
- 把假人声、解释腔、均匀句群洗掉

## 三档返修强度

### 1. 轻手术：皮相拆洗

适用：

- 只有少量套语
- 句子太匀，但情节结构没坏
- 对白发虚，但人物关系还在

动作：

- 把抽象词换成动作、物体、声音、温度、停顿
- 删掉“他知道 / 她明白 / 在这一刻 / 心中涌起”这类作者代说
- 打散连续同长度句群
- 把一部分台词改成动作或反应

不要做：

- 不要改动段落顺序
- 不要替人物补新动机
- 不要把短修做成长改

### 2. 中手术：骨肉重排

适用：

- 解释腔偏重
- 对白承担了过多设定说明
- 人物说话像同一个人
- 情绪只靠抽象词，不靠场面

动作：

- 重排段内重心，让“动作 -> 阻力 -> 反应 -> 后果”先成立
- 给角色补潜台词、试探、遮掩、嘴硬、犹豫
- 把解释句拆进环境反馈、关系压力、代价纹理
- 把同腔对白拆成角色专属口气

不要做：

- 不要改主线结果
- 不要新增大设定
- 不要把场景洗成散文

### 3. 重手术：结构复建

适用：

- 章节大段悬浮
- 情绪与冲突脱节
- 受压无补偿，读感只剩受气
- 结尾只是总结，没有追更驱动

动作：

- 重建“压力 -> 选择 -> 代价 -> 回报/新债”链路
- 重写结尾，让章节停在动作、发现、危险或未决选择上
- 让人物关系或认知状态发生可感知变化
- 回补本章应该兑现的实感回报

不要做：

- 不要把整章洗平为梗概
- 不要为了去味把卖点和爽点一起删掉

## 四类高发病灶

### 1. 套话病

典型信号：

- 在这一刻
- 以 X 为 Y
- 心中涌起
- 仿佛 / 似乎 / 某种感觉 反复连用

修法：

- 换成具体场面
- 让角色碰到东西、看见细节、做出反应

### 2. 解释病

典型信号：

- 旁白帮角色总结情绪
- 对白替作者解释设定
- 连接词堆叠把因果讲穿

修法：

- 把解释改成动作和后果
- 把设定信息埋进冲突代价
- 让角色通过误判、对抗、交易暴露信息

### 3. 均匀病

典型信号：

- 连续 4-6 句长度接近
- 句式都从“他/她 + 动词”起手
- 每段都像同一节奏模板

修法：

- 长短句交替
- 用停顿、插入动作、临场观察打断均匀拍子
- 在关键处把镜头贴近身体和物件

### 4. 同腔病

典型信号：

- 所有人说话语气一样
- 嘴上都很完整、很理性、很会总结
- 没有身份差、关系差、情绪差

修法：

- 给人物加回避、压低、顶嘴、留白、说半句
- 把身份和关系写进措辞选择
- 让不同角色承担不同信息密度

## 返修硬规则

- 不要只把句子修顺，要把“人在场”修出来
- 不要只删词，要补动作、阻力和后果
- 不要把主角承受的压力写成抽象情绪，要写成身体、关系、资源或规则层面的具体代价
- 不要把章节结尾修成感悟，平台向章节结尾优先服务追更
- 去味后复查：开头抓力、中段回报、章末钩子有没有一起被洗掉

## 最短返修流程

1. 先判断是轻 / 中 / 重哪一档
2. 先改最显眼的 1-2 类病灶，不要同时全修
3. 改完复查三件事：
   - 信息有没有丢
   - 人物声音有没有更分化
   - 结尾有没有保住驱动力
