# 定向返修微操作库

用于局部返修、单章返修、选段优化或明确点名问题时，优先选择一个操作定向修，不要一次性并发多个目标把文本改散。

默认原则：

- 一次只解决一个主问题
- 先修结构问题，再修文风问题
- 先保一致性，再追求局部华丽
- 微操作用于定向返修，不替代完整重写流程

## 1. 使用顺序

优先判断问题属于哪一层：

1. 主线/大纲层
2. 章节/场景层
3. 正文表达层

如果问题其实在更高层，不要在低层硬修。

例如：

- “这章没劲” 可能不是文笔问题，而是缺冲突和回报
- “人物突然不对劲” 可能不是对白问题，而是动机没补
- “结尾不想追” 可能不是语言问题，而是前台问题清零了

## 2. 大纲层微操作

### 2.1 深化冲突

适用：

- 大纲有事件，但没有明显对抗
- 主角只是顺着剧情走，没有被真正阻拦

检查点：

- 谁想拿到什么
- 谁在阻止
- 失手代价是什么
- 冲突怎么升级

提示词骨架：

```text
你现在只做一件事：把下面这段大纲的冲突写硬，不重写整体方向。

原始内容：
${target_text}

请补强：
1. 主角此处最明确的行动目的
2. 当前直接阻碍来自谁或什么
3. 失败会失去什么
4. 这一轮冲突如何把局势推向更难的新阶段

输出要求：
- 保留原有主线方向
- 不额外发明大段新设定
- 输出“调整后版本 + 修改说明”
```

### 2.2 补人物动机

适用：

- 行为成立度弱
- 人物像为了剧情硬走位

检查点：

- 这个人为什么现在就要这么做
- 他在怕什么、想守什么、误判了什么
- 换成这个角色的一贯逻辑，会不会选别的

提示词骨架：

```text
你现在只修人物动机，不改主线结果。

目标内容：
${target_text}

相关人设与关系：
${character_context}

请补强：
1. 角色此刻的即时目标
2. 角色的真实顾虑或执念
3. 这个行为为什么符合他的一贯逻辑
4. 如有 角色走样 风险，改成更贴合人设的执行方式

输出“修订版 + 动机解释”。
```

### 2.3 增加伏笔

适用：

- 大纲平推
- 后续转折缺提前铺垫

检查点：

- 伏笔要具体可识别
- 伏笔必须和后续兑现点对应
- 不要只写模糊神秘感

提示词骨架：

```text
请为下面内容增加 1-2 个可回收伏笔，只补铺垫，不抢戏。

当前内容：
${target_text}

后续准备兑现的方向：
${payoff_target}

要求：
1. 伏笔具体，不要泛泛暗示
2. 伏笔最好同时服务人物、关系或世界信息
3. 不改变当前场景主任务

输出“补强版 + 伏笔用途说明”。
```

### 2.4 修正流水账

适用：

- 只有“先发生什么，再发生什么”
- 看不出关键场景和关系变化

提示词骨架：

```text
请把下面这段流水账式大纲，改成可执行的关键帧，不扩成正文。

原文：
${target_text}

每个关键帧至少补齐：
1. 关键事件
2. 行动目的
3. 冲突点
4. 关键场景
5. 关系或认知变化
6. 回报或钩子

输出用列表，不写成小说正文。
```

## 3. 章节层微操作

### 3.1 强化章节冲突

适用：

- 一章有内容但不刺激
- 场景像流程，不像对抗

提示词骨架：

```text
请只强化这一章的即时冲突和代价，不改变章节在全书中的功能。

章节内容：
${target_text}

请补强：
1. 本章最前台的问题
2. 谁在阻止主角
3. 代价和赌注
4. 局部解答后冒出的新问题

输出“修订版 + 本章冲突链说明”。
```

### 3.2 补场景桥接

适用：

- 转折生硬
- 场景像分镜跳切

提示词骨架：

```text
请修复下面内容的转场和因果桥接，不增加无关戏份。

目标内容：
${target_text}

重点补：
1. 上一动作的余波
2. 人物反应
3. 视线/注意力/空间移动的过桥
4. 下一步为何自然发生

输出只给修订版。
```

### 3.3 强化结尾钩子

适用：

- 章节结束后没有追问欲
- 结尾在总结情绪或总结主题

提示词骨架：

```text
请重写这一章结尾，让它留下前台未决问题，而不是替作者收束主题。

当前结尾：
${target_text}

要求：
1. 停在动作、发现、选择或危险上
2. 保留本章已有情绪结果
3. 让下一章有明确承接点

输出“新版结尾 + 钩子类型说明”。
```

### 3.4 增加关系变化

适用：

- 章节有事件，但人物关系像没动过
- 感情线或搭档线只在说话，没有推进

提示词骨架：

```text
请在不改主事件的前提下，让这一段多出明确的关系变化。

目标内容：
${target_text}

相关人物关系：
${relationship_context}

要求：
1. 关系变化要通过动作、选择、让步、误解或信息交换体现
2. 不要靠直接讲道理宣布“关系变了”
3. 保留人物边界，不硬煽情

输出“修订版 + 关系变化说明”。
```

## 4. 正文层微操作

### 4.1 去生成腔

适用：

- 抽象词密集
- 解释腔重
- 情绪和主题都靠直接宣布

提示词骨架：

```text
请给下面内容去生成腔，只改表达，不改剧情信息。

目标内容：
${target_text}

重点处理：
1. 抽象情绪词改成动作、环境、细节
2. 解释腔改成场面内因果
3. 套话和空泛感叹删减

输出只给修订版。
```

### 4.2 优化对白

适用：

- 角色说话一个味
- 台词太书面
- 信息都靠对话硬搬

提示词骨架：

```text
请重写下面这段对白，让角色声音区分更明显。

目标内容：
${target_text}

角色信息：
${character_context}

要求：
1. 每个人说话方式不同
2. 减少互相复述已知信息
3. 用动作、停顿、语气和回避增强攻防感

输出只给修订版。
```

### 4.3 强化感官与环境互动

适用：

- 场景悬浮
- 人物像只在脑内和嘴上推进

提示词骨架：

```text
请增强下面内容的感官细节和环境互动，不额外拖慢节奏。

目标内容：
${target_text}

要求：
1. 增加可触碰、可听见、可看见的具体细节
2. 让人物和环境发生实际接触
3. 细节最好顺手服务情绪或后续功能

输出只给修订版。
```

### 4.4 去说教和总结腔

适用：

- 角色替作者发言
- 段尾和章尾总想升华

提示词骨架：

```text
请删除这段里的说教感和总结腔，但保留原本想表达的内核。

目标内容：
${target_text}

要求：
1. 不直接总结主题
2. 把意思改写进选择、代价、后果或人物反应
3. 结尾不落在概念判断上

输出只给修订版。
```

## 5. 选操作时的禁忌

- 不要同时下“强化冲突 + 增加伏笔 + 强化感情 + 去生成腔 + 润色升华”这种打包命令
- 不要在高层问题没修时，低层反复抛光
- 不要把“润色”当默认动作；默认应先判断结构问题
- 不要让微操作偷改主线、世界规则或核心人设

## 6. 与本仓库工作流的衔接

推荐顺序：

1. 主线问题先看 `references/planning/main-plot-construction.md`
2. 大纲发虚先看 `references/planning/outline-refinement.md`
3. 单章执行看 `references/chapter/chapter-workflow.md`
4. 明确局部问题时，再从本文件挑一个微操作

如果一个问题需要连续调用 2 个以上微操作，说明它大概率不是局部问题，应该回到上层重查。
