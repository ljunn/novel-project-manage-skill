# 审阅报告落盘规则

用于统一 `chapter_pipeline.py review` 生成的章节审阅报告目录和命名，避免审稿稿本只停在终端里。

## 默认行为

- `review` 默认会在项目目录下生成 Markdown 审阅报告
- 默认目录：`<项目目录>/审阅意见/`
- 若已记录当前卷，报告会优先落在：`<项目目录>/审阅意见/<当前卷>/`
- 若未传 `--project-path`，命令会先尝试从章节路径向上自动定位项目根目录

## 默认命名

- 有卷信息：`章节审阅报告_<当前卷>_第XXXX章_<YYYY-MM-DD>.md`
- 无卷信息：`章节审阅报告_第XXXX章_<YYYY-MM-DD>.md`

示例：

- `审阅意见/第一卷/章节审阅报告_第一卷_第0007章_2026-04-22.md`
- `审阅意见/章节审阅报告_第0007章_2026-04-22.md`

## 路径优先级

1. 用户显式传入 `--report-path`
2. 项目根目录 + 当前卷信息
3. 项目根目录 + 默认 `审阅意见/`

## 报告最少应包含

- 章节文件路径
- 静态预审 Verdict
- 字数 / 情绪 / 爽毒点 / 对白统计
- 阻塞问题与风险提示
- 规则命中证据
- 自动预审检查
- 待完成语义审稿问题
- 下一轮返修动作

## 使用建议

- `review` 跑完后先看阻塞问题，再看风险提示，不要先盯局部文笔
- 若命中 `ai_tells / explanation_tone / abstract_psychology`，返修时联动 `references/quality/anti-ai-rewrite.md`
- 阶段治理或批量返修时，优先保留报告文件，不要只看终端输出
