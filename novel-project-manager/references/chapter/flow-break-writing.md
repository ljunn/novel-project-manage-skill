# 流断式创作

用于平台向连载、手机端阅读、番茄系快节奏章节，以及需要降低 生成腔痕迹、增强追读感的动作场、冲突场、情绪爆发场。

它不是“把正文写碎”这么简单，而是：

- 表面上用短流段、高频断句、断点留钩，制造停顿、压迫和追读感
- 内里仍保持视角、情绪、因果和主线目标连续，不让读者读乱

一句话记法：

**一句一段、一段一钩、断在关键、续在情绪、只写当下、不堆废话。**

## 1. 核心定义

流断式创作的核心，不是“短”，而是“短而不断气”。

- **流**：剧情、情绪、视角、行动目的连续
- **断**：在动作节点、对话转折、危险逼近、情绪爆点、真相露角处切段

读感目标：

- 读者翻得快
- 段落压得紧
- 每段末尾都像还有半口气没吐完

## 2. 什么时候用

优先使用场景：

- 开篇前 10%-20% 需要快速抓眼
- 追逐、对抗、揭露、威胁、争吵、翻盘
- 强情绪场：羞辱、失控、心碎、恐惧、愤怒、欲望上头
- 去生成腔时，原文明显过长、过顺、过像说明文
- 手机端阅读压力要降下来时

谨慎使用场景：

- 世界规则解释
- 复杂博弈推理
- 情绪回落后的收束段
- 需要稳定铺陈空间关系或信息关系的场景

原则：

- 它适合“提速”和“加压”
- 不适合把整章所有段落都切成同一种碎句节奏

## 3. 实操标准

### 3.1 极短流段

- 单段通常 1-3 行
- 单段通常只承载 1 个信息点
- 一个信息点可以是：
  - 一个动作
  - 一句对白
  - 一个眼神或身体反应
  - 一个短促心理闪念
  - 一个画面切换

避免：

- 一段里同时塞动作、解释、背景、感悟、设定、关系判断
- 一口气写完整因果链，读起来像汇报

### 3.2 高频断句

- 优先用句号、问号、感叹号切开
- 少让长逗号链一路拖到底
- 能拆成两句，就不要硬拖成一整句

断句目标：

- 节奏有停顿
- 每一句都像往前顶一步
- 不让语气匀速滑过去

### 3.3 断点留钩

优先断在这些位置：

- 动作刚起，还没落地
- 对话只说了一半
- 危机刚被看见
- 真相只露一角
- 情绪刚炸开
- 选择还没做完

段尾优先落点：

- 未完成动作
- 新风险
- 新发现
- 反问
- 认知拐点

不要总落在：

- 总结句
- 感悟句
- 解释句
- 作者替角色收尾的判断句

### 3.4 形散神聚

流断式的底线是：表面碎，内里不能散。

必须保持：

- 主视角稳定
- 当前主问题稳定
- 情绪递进清楚
- 动作之间有余波、有反应、有因果桥

如果只是把段落切碎，但看不出：

- 为什么现在断
- 下一段为什么接得上
- 主角此刻到底想做什么

那就不是流断式，只是碎。

## 4. 和 生成腔净化的关系

流断式对下面这些问题很有效：

- 长句过多
- 长段过多
- 解释腔太重
- 抒情像统一模板
- 节奏匀速，像说明书

但它不是万能药。

如果问题是：

- 剧情本身没冲突
- 人设动机不清
- 视角越权
- 设定逻辑没立住

那先修结构，不要只靠切短句救稿。

## 5. 常见误区

### 5.1 只有断，没有续

症状：

- 每段都像独立短视频字幕
- 场面之间没有余波
- 情绪跳来跳去

修法：

- 给下一段补“反应层”或“因果层”
- 让上一动作留下的疼、热、响、视线、姿势，带进下一段

### 5.2 只有碎，没有钩

症状：

- 段很短，但没有紧张感
- 每段都只是拆句，不是在推进

修法：

- 每段问自己：这一段到底是推进动作、抬升风险，还是制造期待？
- 如果都不是，就删或并回上一段

### 5.3 全章同速

症状：

- 从头到尾都一个节奏
- 读久了像刻意制造“网感”

修法：

- 冲突场用流断式
- 解释段、回落段、关系沉淀段适当放长
- 让节奏像心跳，不要像打点器

### 5.4 机械问句泛滥

症状：

- 连续用“疼吗？”“为什么？”“她懂吗？”这类问句刷存在

修法：

- 问句只留最有力的那个
- 其他改成动作、停顿、环境反馈

## 6. 自检清单

- [ ] 这一段是不是只承载一个核心信息点
- [ ] 段尾是不是停在动作、风险、发现、选择或情绪拐点上
- [ ] 下一段是不是接得上前一段的动作余波或情绪惯性
- [ ] 主视角有没有被切碎到失焦
- [ ] 有没有为了“像真人”而故意切得过碎
- [ ] 如果把这些短段并回去，冲突是否会明显变钝

## 7. 适合放进技能的定位

建议定位：

- 作为平台向正文的**可选节奏技法**
- 作为“去生成腔 / 提升留存”的**场景级工具**
- 不是所有题材、所有章节、所有段落都必须套用的统一模板

和本仓库其他规则的关系：

- 服从视角护栏
- 服从连贯性检查
- 服从“正文像连续水流，不像任务清单”的底线
- 优先服务钩子、代价、关系变化和追读牵引
