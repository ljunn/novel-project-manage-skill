# Technique Stack

Use this index before drafting or revising a chapter. `scripts/chapter_pipeline.py compose` auto-recommends a compact stack from the chapter title, current goal, active foreshadowing, and guidance. Override only when the current problem clearly needs a different card.

## Default Every Chapter

Always keep these active:

- `references/chapter/writing-quickref.md`: chapter/scene/language baseline.
- `references/chapter/chapter-workflow.md`: pre-read, scene split, pure manuscript rules.
- `references/chapter/chapter-template.md`: what not to put into manuscript files.

## Choose By Chapter Need

- `references/chapter/hook-techniques.md`
- `references/chapter/literary-opening.md`
- `references/chapter/suspense-design.md`

- `references/chapter/descriptive-taxonomy.md`

## Technique Stack Output

Before drafting, write a compact stack like this into the working notes, not the manuscript:

```text
Technique Stack:
- Baseline: writing-quickref + chapter-workflow
- Main technique: hook-techniques
- Support technique: reader-compensation
- Risk guard: rule-linting
```

Then draft prose normally. The reader should never see technique labels, beat labels, or scene-card headings.
